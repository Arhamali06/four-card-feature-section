# Frontend Mentor | Four card feature section

This is my solution to Frontend Mentor's [Four card feature section challenge](https://www.frontendmentor.io/challenges/four-card-feature-section-weK1eFYK).

## Overview

The challenge is to recreate a feature section containing four cards. Each card has a coloured top border, a title, supporting text, and an icon.

### The challenge

Users should be able to:

- View the four feature cards on desktop and mobile screen sizes.
- See the cards arranged around the central pair on larger screens.
- See the feature information, icons, and coloured card accents clearly.

### Links

- Solution repository: [GitHub](https://github.com/Arhamali06/four-card-feature-section)
- Live site: Not deployed yet.

## Built with

- Semantic HTML5
- CSS
- Flexbox
- Responsive CSS media queries

## What I learned

While building this project, I practised:

- Structuring a page with headings, paragraphs, cards, and images.
- Using Flexbox to place cards in rows and columns.
- Using `flex-wrap` to allow cards to move onto a new line when there is not enough room.
- Adding coloured borders and box shadows to create card depth.
- Positioning icons inside cards with Flexbox.
- Using media queries to adapt text and layout for smaller screens.

## Continued development

In future projects, I want to keep improving at:

- Building mobile layouts first and testing a wider range of screen sizes.
- Reducing repeated CSS by sharing common card styles.
- Using the exact fonts, font weights, and spacing from a design system.
- Improving semantic HTML and accessibility, including descriptive image alternative text.
